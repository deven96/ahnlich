# Stores

A **store** is Ahnlich's main unit of organization — a named container for your
vectors and their metadata, much like a *collection* in a document database or a
*table* in a relational one. You create a store, insert data into it, then query
it by similarity or by metadata predicates.

Every operation below is shown in **Python, Node.js, Go, Rust, and the CLI** —
pick your language once and it stays selected across pages.
